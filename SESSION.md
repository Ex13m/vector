# Сессия: v0.5.13 → v0.5.18

## Что делали

### 1. Стрелка на PickScreen смотрела на юг
**Проблема:** При запуске карта стояла «югом вверх» — стрелка «я» визуально смотрела вниз (на юг), хотя должна всегда смотреть вверх (на север).

**Что пробовали:**
- Сначала ошибочно повернули саму стрелку на 180° — неверно, надо было карту.
- Потом заблокировали вращение карты (`dragRotate.disable()`) — пользователь спросил «почему крутить нельзя?», убрали.
- В итоге: `bearing: 0` в конструкторе + `setBearing(0)` в событиях `load` и `styledata` + `bearing: 0` в `flyTo`.

**Результат:** Карта стартует на севере, стрелка смотрит вверх. ✓

---

### 2. Мини-компас на PickScreen
**Задача:** Добавить виджет компаса, который вращается от `deviceorientation`.

**Реализация:**
- Круглая кнопка 44×44 справа под кнопкой настроек.
- Внутри — стрелка с «N» вверху, красный штырь на север, серый на юг.
- Поворачивается через `transform: rotate(${-compassHeading}deg)`.
- iOS: клик на виджет запрашивает разрешение `DeviceOrientationEvent.requestPermission()`.

---

### 3. Компас дёргался
**Проблема:** Сырые события `deviceorientation` вызывали дрожание стрелки.

**Исправление:**
- Экспоненциальный LPF: `smoothed += 0.12 * diff` с обработкой перехода 0°/360°.
- Throttle через `requestAnimationFrame` — обновление не чаще 60fps.

**Результат:** Стрелка плавная. ✓

---

### 4. Карта в RideScreen двигалась рывками
**Проблема:** `jumpTo` вызывался раз в секунду при каждом GPS-фиксе — видимые рывки.

**Было:**
```
jumpTo(center) — 1 Hz по GPS
easeTo(bearing, 200ms) — 16 Hz по компасу
```
Два конкурирующих эффекта прерывали друг друга.

**Стало:** Единый rAF-цикл 60fps с экспоненциальным лерпом:
- Позиция: `k = 0.12` — плавное скольжение к GPS-точке.
- Курс: `k = 0.18` — плавный доворот камеры.
- Оптимизация: пропуск `jumpTo` когда позиция/курс не изменились (экономия GPU).

**Результат:** Карта движется плавно. ✓
