# APK v0.5.45 — баги полевого теста

Список проблем при тестировании APK. Дебажим после сбора всего списка.

## Открытые

1. **Нет лока ориентации** — APK вращает экран при повороте телефона. Должен быть залочен (как в PWA, где стоит запрет поворота). Нужно залочить ориентацию в portrait на уровне Capacitor/AndroidManifest.

2. **Компас съехал вверх (safe-area) — ПОДТВЕРЖДЕНО v0.5.46** — причина: targetSdk 36 (Android 15) принудительно делает edge-to-edge, но Android WebView НЕ отдаёт `env(safe-area-inset-top)` в CSS (возвращает 0). Наш CSS рассчитывает отступы через `env(safe-area-inset-*)` → всё уезжает под статусбар. Юзер описал точно: "нет рамки сверху и снизу как в PWA". ФИКС: плагин `@capacitor-community/safe-area` пробрасывает реальные insets в `env()`, существующий CSS заработает.

3. **Нет начального позиционирования (первый экран)** — при открытии нет стрелки "вы" в центре, нет позиции пользователя → карта показывает неправильный масштаб/зум. Вероятно GPS-фикс не приходит на старте (нативный плагин требует выдачи разрешения + первый фикс занимает время; начальный вид карты не центрируется на юзере).

4. **Артефакты внизу справа** — под кнопкой непонятные визуальные артефакты. Уточнить что за элемент (возможно MapLibre attribution / logo / safe-area наложение).

5. **Нет иконки центрирования** — отсутствует кнопка/иконка recenter (вернуть карту на позицию пользователя).

6. **Нет голоса (ПОДТВЕРЖДЕНО)** — TTS молчит и в помещении, и на улице. Везде. GPS при этом работает. GPS при этом работает (позиция приходит). Нативный `@capacitor-community/text-to-speech` не произносит. Проверить: установлен ли системный TTS-движок (Google TTS), вызывается ли `speakNative`, не падает ли в catch. Возможно нужна инициализация/выбор движка или язык не поддерживается устройством. ВАЖНО: это core-фича. _(не дотестировано до фонового режима)_

7. **Не видно где своя позиция → непонятно как выбрать цель (PickScreen)** — нет маркера "вы" на первом экране, поэтому юзер не понимает откуда он и как выбрать цель относительно себя. Связано с багом №3 (нет начального фикса/центрирования). Карта не центрируется на пользователе, маркер позиции не отображается. Критично для UX выбора цели.

8. **Карта открывается на дефолтной позиции (Москва, z4)** — при старте карта показывает дефолтный вид (Москва, зум 4) вместо центрирования на пользователе. Подтверждает корневую причину багов №3/№7: начальный GPS-фикс не применяется к камере карты при старте. Вероятно `getLastKnownPos()` пустой и нет ожидания первого фикса от нативного плагина перед `flyTo`/`setCenter`. КОРНЕВОЙ баг для №3, №7.

9. **CacheScreen — медленный старт (тот же симптом что №3/№8)** — экран кэширования области тоже долго грузит масштаб/центрирование с вектором. Та же причина: медленный GPS-фикс. ДОЛЖНО быть исправлено в v0.5.46 (CacheScreen переведён на `@capacitor/geolocation`). ПРОВЕРИТЬ: тестировал ли юзер v0.5.46 или старую сборку?

10. **RideScreen PRE_RIDE — медленный GPS-старт (НЕ исправлено в v0.5.46)** — на главном экране поездки тоже долго ловит позицию/центрирование в помещении. Причина: RideScreen использует `@capgo/background-geolocation` (фоновый, `stale:false`) — он медленно даёт первый фикс. Я перевёл на быстрый GPS только Pick/Cache, RideScreen НЕ трогал. ФИКС: засеять начальную позицию быстро — `getLastKnownPos()` сразу + быстрый первичный фикс через `@capacitor/geolocation`, потом передать управление фоновому плагину для трекинга. Либо разрешить `stale: true` для первого фикса.

## Гипотезы по корневым причинам

- **Safe-area / fullscreen (баги 2, 4, 5, и часть верстки):** в PWA контент рисуется внутри вьюпорта браузера (учитывает статусбар), в APK — fullscreen edge-to-edge под статусбар/вырез. Элементы у краёв заезжают наверх. Фикс: `@capacitor/status-bar` + `viewport-fit=cover` + `env(safe-area-inset-*)` в CSS. _(гипотеза пользователя: "в PWA есть рамка экрана, в APK нет" — подтверждает это)_
- **Смена GPS-провайдера (баги 3, 7, 8) — ПОДТВЕРЖДЕНО скринами:** на PickScreen `me` остаётся `null` — нативный `@capgo/background-geolocation` не доставляет позицию в React-стейт. Каскад: маркер "вы" (gated на `me`), компас-циферблат, кнопка recenter, центрирование карты — ВСЁ это завязано на `me`, поэтому в APK ничего нет, карта стоит на дефолте Москва z4. В PWA `navigator.geolocation` отдаёт фикс сразу → всё работает.
  - **Сравнение скринов:** PWA — стрелка "вы" в центре, компас справа вверху, кнопка-прицел справа внизу, реальный z15. APK — ничего из этого, только attribution "i" внизу справа (= "артефакт" бага №4).
  - **УТОЧНЕНО:** фикс ВСЁ-ТАКИ приходит, но очень медленно в помещении. На улице заработало сразу. Причина — в обёртке стоит `stale: false` → плагин ждёт настоящий GPS-фикс, не отдаёт кэш/сетевую позицию. Браузер в PWA отдаёт сетевую позицию мгновенно.
  - **Фикс (план):**
    1. Показывать `getLastKnownPos()` сразу при старте (если есть) — не висеть на Москве.
    2. Для PickScreen/CacheScreen: `stale: true` ИЛИ добавить `@capacitor/geolocation` для быстрого первичного фикса (сетевая позиция), а `@capgo/background-geolocation` оставить ТОЛЬКО для RideScreen (фон).
    3. Пока нет фикса — показывать индикатор "ищу GPS…" вместо тихого дефолта Москва.
