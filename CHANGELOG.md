# История изменений

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), нумерация — [Semantic Versioning](https://semver.org/lang/ru/).

## [0.4.3] — 2026-05-12

### Исправлено

- **Маркер цели всё ещё не совпадал с концом вектора** после v0.4.2: `display:flex` поверх `position:absolute` ребёнка иногда даёт нестабильное центрирование (зависит от браузера и от того, в каком порядке считается layout). Перешёл на **жёсткое позиционирование**: контейнер маркера 1×1 px (anchor='center' попадает строго в гео-точку), каждый дочерний элемент позиционирован абсолютно с `left:50%; top:50%; margin:-W/2 0 0 -H/2`. Это гарантирует, что и пульс-кольцо, и иконка центруются ровно на гео-точку. Поправлено для маркеров `вы` и `цель` на всех экранах (Pick / Cache / Ride / ArrivedOverlay).
- **На экране Arrived не было оранжевого вектора «последняя точка трека → цель»** — теперь рисуется пунктиром в мини-карте. Также добавлен зелёный пунктирный трек (был сплошной линией). Маркеры и fitBounds создаются сразу, не внутри `map.on('load')` — чтобы появлялись даже если спутниковые тайлы не успели загрузиться.

## [0.4.2] — 2026-05-12

### Исправлено

- **Маркер цели не совпадал с концом оранжевого вектора** (видно на 60 км в превью кэша). Pulse-кольцо имело `position:absolute` без `inset:0`, контейнер был меньше пульса по размеру — браузер ставил кольцо в угол. Контейнер маркера теперь равен размеру пульса, кольцо стоит `inset:0`, иконка центрируется флексом, MapLibre `anchor: 'center'` указан явно. Поправлено на Pick / Cache / Ride. _(Помогло частично; полностью исправлено в v0.4.3.)_
- **Кнопка «Пропустить» на CacheScreen едва видна** на спутнике — был прозрачный фон + `inkDim` текст. Теперь полупрозрачный тёмный фон с blur, белый текст, тактильная отзывчивость на тап.

## [0.4.1] — 2026-05-12

Исправления по реальной поездке: пользовательские багрепорты v0.4.0.

### Исправлено

- **Слайдер интервала голоса делал шаг 1 сек** вместо 1 мин. Причина — циклический импорт `App.tsx ↔ SettingsSheet.tsx`: при первом чтении `VOICE_INTERVAL_STEP` был `undefined`, и `<input type="range">` подбирал дефолтный step=1. Константы вынесены в `src/lib/constants.ts`.
- **Оранжевый вектор «вы → цель» криво соединялся с маркерами** на экранах Pick и Cache (CSS-overlay подходит для статики, но при наклоне расходился с MapLibre-проекцией маркеров). Переведено на MapLibre GeoJSON-source — пиксель-в-пиксель.
- **Пилюля расстояния** теперь Marker на геогр. середине отрезка — следует за линией при панорамировании и зуме.
- **Цели 50+ км не кэшировались** — фиксированный диапазон zoom 12-16 упирался в лимит 2000 тайлов. Заменено на адаптивный диапазон: 3 зум-уровня вокруг текущего зума карты («что видишь — то и кэшируется»).
- **На экране Ride не было видно точки куда едешь** при дистанции 50+ км. При первом GPS-фиксе теперь делается `fitBounds(me, target)` если до цели >800 м; иначе по-прежнему flyTo на пользователя.
- **На экране Arrived не было маркера конечной точки** — добавлена мини-карта 180px над сводкой: спутник, зелёный трек пройденного пути, зелёная точка старта, оранжевый прицел цели, fitBounds на все три. Большая галочка ✓ перемещена в угловой бейдж.
- **Иконки слоя / мини-циферблат / LIVE бейдж гасли через 5 сек** до opacity 0.25. Auto-hide chrome полностью удалён по запросу.
- **«Новая цель» — флоу был корректным**, но без тактилки тап не ощущался. Добавлен haptic — обратная связь теперь есть.

### Добавлено

- **Тактильная отзывчивость** — новая утилита `src/lib/feedback.ts` с паттернами `light`/`medium`/`heavy`/`success`. Подключена к ключевым кнопкам:
  - Pick: «Старт», ★ сохранить, тап подсказки поиска, тап карты для постановки цели.
  - Cache: «Сохранить область», «Отмена».
  - Ride: тулбар (Pause · Voice · Mute · Stop).
  - Arrived: «Журнал», «Новая цель», «↩ Вернуться к старту».
- **Звуковой сигнал «цель впереди»** — короткий двойной beep `A5 → E6` через WebAudio при переходе clockNum → 12 (плюс усиленный haptic). Реализовано в `feedback.ts:chimeOnTarget()`. Не звучит при `mute`.
- **Лейблы на спутнике** — поверх Esri World Imagery теперь рисуется второй raster source с Esri «Boundaries and Places» (бесплатный), города и крупные дороги подписаны.
- **Дефолтный слой карты — спутник** (был CartoDB Dark).

### Изменено

- Workbox runtime-cache `map-tiles` поднят до 8192 entries (было 4096). Добавлены хосты `server.arcgisonline.com` (Esri спутник + лейблы) и `tile-cyclosm.openstreetmap.fr` (велослой).

## [0.4.0] — 2026-05-12

Большая переработка по эталонному дизайн-бандлу `Screens Explainer.html` + фоновый голос (главный пользовательский блокер).

### Добавлено

- **Фоновый голос при заблокированном экране**. Новый модуль `src/lib/wakeAudio.ts`: тихий `AudioContext`-loop + `MediaSession` API. Делает вкладку «media-active», и `speechSynthesis` не глохнет, `setInterval` не троттлится. Обработчик `visibilitychange`: при пробуждении устройства отменяет накопившуюся очередь речи и ждёт свежего GPS-фикса перед следующей фразой — больше не озвучивает устаревшие данные.
- **Шаг циферблата 5 минут**: `relativeToClockHM` в `geo.ts` теперь возвращает строку вида `«12:05» / «11:50» / «12:55»` с гранулярностью 2.5° (144 сектора). Прежний 1-часовой расчёт `relativeToClock` сохранён для подсказок.
- **Расширенная фраза голоса**: `«Цель на 11 часов 45 минут, 2 километра 400 метров, ехать 14 минут»` (RU/EN/DE). ETA включается, когда оценка надёжна.
- **Гранулярный геокодер**: `searchPlace` отдаёт `{name, context, lat, lng}` для отдельной подсветки совпадения и подзаголовка. Добавлен `reverseGeocode(lat, lng)` для названия места по координатам цели.
- **Журнал поездок из Arrived**: кнопка «Журнал» перенаправляет на Pick и сразу открывает Saved-sheet на табе «Поездки». Полный круг: Arrived → Журнал → ▶ Продолжить → Reverse-Ride.

### Изменено

- **Экран 01 Pick** полностью переработан. Компактный topbar (поиск + popover слоя + ★) вместо большого хедера «Where to?». Подсказки появляются внутри капсулы поиска по мере ввода с debounce 400 мс и оранжевой подсветкой совпадения. Маркер «вы» — зелёная ромб-стрелка с поворотом на цель. Маркер цели перетаскивается долгим тапом + drag. Карточка снизу показывает название места из reverse-geocode.
- **Экран 02 Cache** полностью переработан. Full-bleed карта без рамок. На входе — `fitBounds(me, target)` с padding 12%. Top-card с live-счётчиком тайлов / МБ / диапазона zoom. Pinch меняет область кэширования. Лимит 2000 тайлов — красная карточка, disabled CTA. Авто-скип в Ride если все тайлы уже в кэше (проверка subdomain a/b/c).
- **Экран 03 Ride** полностью реструктурирован. LIVE-бейдж → красный «GPS LOST» через 10 сек без фикса. Тулбар упрощён до 4 кнопок (Pause · Voice · Mute · Stop) — метрики SPEED/RIDDEN/TIME перенесены на Arrived. HUD на 3 ячейки: TO TARGET · AT O'CLOCK (теперь H:MM) · ETA. Маркер «вы» — двухрежимная ромб-стрелка: в движении (>0.5 м/с) поворачивается по вектору двух последних GPS-точек, на стоянке переключается на магнитометр. Трек зелёным пунктиром + оранжевый пунктир «вы → цель» (нативные MapLibre line layers). Pause останавливает запись трека, не только голос. Кнопка «Назад» через `history.pushState`/`popstate` → модалка «Продолжить или Завершить?». На Arrived back ведёт сразу на Pick без вопросов.
- **Экран 04 Arrived** перестроен. Автосохранение поездки в момент триггера (было — только по кнопке). Голос «Вы у цели» (RU/EN/DE). Имя по умолчанию `«Поездка от dd.mm hh:mm»`. Сводка 4 ячейки: Время · Дистанция · Средняя · **Макс. скорость** (новое). Три кнопки: «Журнал» / «Новая цель» / `↩ Вернуться к старту`.
- **ETA** теперь считается по средней скорости всей поездки (`distRidden / timeRidden`), не последних 60 секунд. Первые 30 сек — прочерк `—`. На длительной стоянке (`avgSpeed → 0`) ETA замораживается на последнем валидном значении.
- **Settings**: интервал голоса теперь 0–15 мин с шагом 1 мин (было 0–10 мин шагом 10 сек). Дефолт — 15 мин. Ключ хранилища поднят `vector.settings.v2 → v3` (старые значения с лимитом 600 сек не перетирают новый дефолт).

### Технические детали

- Анимация `liveBlink` добавлена в `globals.css`.
- `.claude/` добавлен в `.gitignore` (локальные agent-настройки).

## [0.3.0] — 2026-05-12

Полная переписывание экранов под скриншоты дизайна. Без новой функциональности, но визуал и UX подгоняются под исходный референс.

### Изменено

- **PickScreen**: clock-position навигация в верхнем баре, геокодер с поиском по Enter, постоянная панель слоёв справа, sheet с сохранёнными точками/поездками, flyTo zoom 16 при первом GPS-фиксе.
- **CacheScreen**: реальный prefetch тайлов с прогрессом, слайдер delta-зума, поддержка отмены через `AbortController`.
- **RideScreen**: курс по вектору последних двух GPS-точек (`bearingFromTrail`), MiniDial (54×54) + BigDial fullscreen-peek через long-press, голосовой цикл, overlay прибытия.

### Исправлено

- `UpdateToast` всегда показывался: `needRefresh` — это массив `[boolean, setter]`, не сам флаг. Распакован корректно.

### Добавлено

- Новые модули: `geocoder.ts`, `mapStyles.ts`, `storage.ts` (IndexedDB через idb-keyval), `voice.ts` (Web Speech API).

### Удалено

- Неиспользуемые компоненты из v0.2 каркаса: `BigClockDial`, `BottomHud`, `ClockDial`, `LayerPicker`, `StatusPill`, `i18n.ts`, `speech.ts`.

## [0.2.0] — 2026-05-05

Полная пересборка прототипа в production-стек. Из дизайн-референса — в рабочий PWA.

### Добавлено

- **Vite 5 + React 18 + TypeScript** вместо Babel-в-браузере. Бандл собирается, минифицируется, code-splittится.
- **Service Worker** через `vite-plugin-pwa` (Workbox): precache app shell, runtime-кэш для шрифтов и тайлов карт, навигационный fallback.
- **Тост обновления**: `useRegisterSW({ onNeedRefresh })` → пользователь видит «Доступно обновление · ОБНОВИТЬ», по клику SW активируется и страница перезагружается.
- **Установка PWA**: баннер «Установить Vector» по событию `beforeinstallprompt` (Android) и инструкция «Поделиться → На экран Домой» (iOS) с откладыванием на 7 дней.
- **MapLibre GL JS** на всех трёх экранах. Стили: OSM Standard, Esri World Imagery (спутник), OpenTopoMap, Waymarked Trails. Растровые источники с тёмным `raster-saturation` и приглушённой контрастностью.
- **Реальная геолокация** (`Geolocation.watchPosition`, `enableHighAccuracy`) — позиция, скорость, аппаратный heading, точность.
- **Ориентация устройства** (`DeviceOrientationEvent.absolute` или `webkitCompassHeading` для iOS) с экспоненциальным сглаживанием 0.25 и запросом разрешения для iOS 13+.
- **Клавишный голос** (`SpeechSynthesis`): фраза «Цель на N часов, X км» каждые `intervalSec` (по умолчанию 15 мин). Кнопка sayNow в HUD. Можно выбрать конкретный голос в настройках.
- **Хаптика** (`navigator.vibrate`): короткий импульс при смене «часа», тройной паттерн при попадании в 12 часов.
- **Офлайн-кэш карты**: расчёт BBox + zoom levels, последовательная закачка тайлов в IndexedDB через `idb-keyval` с конкурентностью 6, прогресс-бар, отмена.
- **Сохранение** избранных точек (до 50) и поездок (до 100) в IndexedDB.
- **Локализация RU/EN** через `i18n.ts` со словарями и подстановкой переменных. Авто-выбор по `navigator.language`.
- **Auto-hide chrome**: top bar и floating-кнопки тают до opacity 0.18 через 5 секунд бездействия. Тап по карте — снова видны.
- **Long-press peek**: 380 ms на mini-циферблате → fullscreen ClockDial 280×280 с цифрами часов, авто-закрытие через 2.2 с.
- **Near-target glow** (`<500 м`): нижний HUD меняет фон/glow/акцентный цвет с оранжевого на зелёный, transition 400 ms.
- **Arrived overlay**: при подходе ближе 50 м — экран финиша с результатами, инпутом названия и сохранением в IndexedDB.
- **Settings sheet**: интервал голоса (slider 0–1800 step 60), единицы (метрические/мили), хаптика, язык, выбор голоса.
- **Reverse-курс**: режим «вернуться назад» — цель становится стартом, цель ‒ исходная точка пользователя.
- **CHANGELOG.md** и обновлённый **README.md** с инструкцией для разработчика и пользователя.

### Изменено

- Прототип целиком перенесён в папку `prototype/` как референс.
- Иконки PWA — отдельные файлы `any` и `maskable` в `public/`.
- `netlify.toml`: build = `npm run build`, publish = `dist`, SPA-fallback, корректные `Cache-Control` для `sw.js` / `manifest`.

### Удалено

- Babel standalone и React UMD из `index.html` — теперь нормальный ESM-бандл.
- `android-frame.jsx` и `tweaks-panel.jsx` — это были инструменты прототипа.
- Симулятор движения с random-drift bearing — заменён реальным GPS.

## [0.1.0] — 2026-05-05

Первый деплой — статический HTML-прототип из дизайн-хэндоффа.

### Добавлено

- HTML-прототип `Vector.html` с JSX через Babel-в-браузере.
- Три экрана: Pick / Cache / Ride.
- Дебаг-панель Tweaks (язык, симуляция, экран, слой).
- Иконки PWA, manifest, простой service worker (precache + cache-first).
- Тост «Доступно обновление».
- `netlify.toml` для статической раздачи прототипа.
- README с описанием прототипа.
- Деплой на Netlify через GitHub.

### Известные ограничения

- Геолокация **не использовалась** — позиция симулировалась случайным дрейфом.
- Карта была **нарисованной декорацией**, без реальных тайлов.
- Голос — только в спецификации.
- Поездки в памяти вкладки, без сохранения.
