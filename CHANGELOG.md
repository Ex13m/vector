# История изменений

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), нумерация — [Semantic Versioning](https://semver.org/lang/ru/).

## [0.5.13] — 2026-05-15 — стабильная версия

### Добавлено

- **Номер версии в настройках** — внизу панели Settings показывается `Vector v0.5.13` (из `package.json` через Vite `define`). Помогает на глаз сверить, какой билд реально загружен в PWA — раньше после нескольких деплоев было не понять, обновился ли service worker.

### Исправлено

- **Ориентация карты в PRE_RIDE: коррекция компаса −90°**. На устройстве пользователя `deviceorientationabsolute` выдаёт heading со смещением на 90° против часовой — карта недокручивала четверть оборота. В `courseHeading` для PRE_RIDE/LONG_STOP добавлена коррекция `+270°` (= −90°). RIDING (GPS-bearing) не затронут — там курс корректен.
- **Прыжок ориентации карты при старте RideScreen**. Карта создавалась с `bearing:0` и `easeTo`-анимацией 200 мс доворачивала к реальному курсу — пользователь видел промежуточное положение. Теперь карта создаётся сразу с прогретым bearing, а первый `setBearing` — мгновенный (без анимации от 0°).
- **Прогрев компаса передаётся в RideScreen**. LPF-состояние компаса вынесено в общий модульный `_sharedSmoothed` (`orientation.ts`): прогрев в `App.tsx` накапливает значение, RideScreen стартует с уже сглаженного показания через `getLastHeading()` — нет скачка фильтра от нуля.
- **CacheScreen: модалка «Тайлы в кэше» не появлялась для реальных маршрутов**. Авто-скип проверял только первые 200 тайлов с условием `total ≤ 200` — для маршрута >200 тайлов модалка не показывалась никогда. Теперь проверяется равномерная выборка из полного набора (`sampleTiles`), условие по размеру убрано. Проверка запускается с задержкой 700 мс — после того как `fitBounds` и пересчёт тайлов устаканились (раньше проверялась область до fitBounds).

## [0.5.12] — 2026-05-14

### Добавлено

- **POI-поиск с иконками**: Nominatim теперь ищет магазины, кафе, заправки и т.д. (всё что есть в OSM). Результаты помечены эмодзи-иконкой по типу (🛒 супермаркет, ☕ кафе, ⛽ заправка, 💊 аптека, 🍽️ ресторан, 🚉 станция и т.д.). Адреса без иконки — стандартный pin.
- **Location bias в поиске**: передаётся `viewbox` ~50 км вокруг текущей GPS-позиции. Ближайшие результаты показываются первыми — «Сільпо» покажет ближайший, а не из другого города. Лимит результатов увеличен с 5 до 8.
- **Map-rotation navigation** (стиль Google Navigation): маркер «вы» фиксирован ↑ вверх, карта вращается вокруг маркера по `courseHeading` (компас в PRE_RIDE, GPS-bearing в RIDING). Убрана вся логика CSS rotate стрелки (arrowRotRef, аккумулятор, 0/360 workaround). Один координатный мир — MapLibre делает rotation нативно, плавно, без багов. Position через `jumpTo` (1 Hz GPS), bearing через `easeTo` duration:200 (16 Hz компас) — плавная интерполяция MapLibre берёт shortest path сам. Deadzone 1° для bearing — экономит CPU/батарею на микро-дельтах.
- **Lock view 🔒 / 🔓**: кнопка блокировки свободного просмотра карты (слева от центровки). Заблокированы pan/rotate, оставлен только zoom. При активации — автоцентровка включается. SVG-иконка в стиле приложения, зелёная подсветка в активном состоянии.
- **Zoom badge**: текущий уровень зума тайлов (Z14, Z15…) отображается слева сверху под layer-кнопкой. Цвет — оранжевый `C.target` для контраста, моноширинный шрифт с tabular-nums чтобы не дёргался.
- **Кнопка центровки всегда видна** (при наличии GPS) — подсвечивается оранжевым когда autoFollow выключен. SVG crosshair-иконка вместо `⊕`. При нажатии — мгновенный `jumpTo` к маркеру + bearing (без race condition с easeTo вращения).

### Изменено

- **Вращение карты разблокировано** на RideScreen и CacheScreen. Раньше `dragRotate.disable()` + `disableRotation()` блочили любой поворот. Теперь bearing карты отслеживается реактивно (`map.on('rotate')` → state), стрелка компенсирует `courseHeading − mapBearing` при каждом изменении обоих значений. Два пальца крутят карту — стрелка остаётся на векторе.

### Исправлено

- **Стрелка «вы» в PRE_RIDE не докручивала** на 0.5°–1° при остановке вращения. Причина: `MIN_DELTA_DEG = 1°` блокировал эмит на хвосте экспоненциальной сходимости LPF. Гейт по дельте убран совсем, throttle оставлен только по времени (16 Hz). LPF сходится до численного нуля, CSS-transition визуально сглаживает мелкие шаги.
- **Стрелка дёргалась при переходе через северный 0°** (359° → 1° = CSS интерпретировал как −358° и крутил «обратно» почти на полный оборот). Заведён аккумулятор `arrowRotRef`: накапливаю поворот непрерывно через shortest-path delta `(((target − current) % 360) + 540) % 360 − 180`. Теперь стрелка всегда идёт кратчайшим путём.
- **Маркер цели на CacheScreen не совпадал с концом вектора**. Старый маркер — 32×32 контейнер с SVG (`left:4px;top:4px`) и pulse-ring через `inset:-6px`. Sub-pixel rounding + анимация ring'а двигали визуальный центр на 1–2 px относительно WebGL-конца линии. Заменён на простой solid-круг 20×20 (как в RideScreen) — один DOM-элемент без вложений и анимаций bounding box.
- **Компас отзывчивее в PRE_RIDE-режиме**: α фильтра 0.25 → **0.5** (тот же шаг сглаживания за 2 фрейма вместо 5), эмит 10 Hz → **16 Hz** (60ms интервал). При интерактивном вращении телефона ощущается как настоящий компас, не как «GPS-стрелка с инерцией».
- **Map rotation заблокирован** на RideScreen и CacheScreen (`bearing:0` + `dragRotate.disable()` + `touchZoomRotate.disableRotation()`). Случайный двупальцевый жест больше не вращает карту → вектор-линия (рендерится в WebGL) и стрелка (CSS-rotate) всегда выровнены.
- **Haptic-спам в PRE_RIDE/LONG_STOP убран** — `haptic('light')` срабатывал на каждое изменение часовой позиции при вращении телефона. Теперь только в RIDING.
- **On-target trigger: instant edge-detection без dwell** во всех фазах. **±2° trigger + ±5° hysteresis**. Каждое пересечение вектора (вход в ±2°) мгновенно даёт голос/chime + haptic. Гистерезис ±5° перезаряжает — надо выйти за 5° чтобы снова сработало. Сценарий «вожу телефоном туда-сюда» теперь работает: каждое прохождение через цель = сигнал. Раньше 200ms dwell в PRE_RIDE/LONG_STOP блокировал быстрые проходы. TargetingHud визуальная зона ±2°.
- **CacheScreen: auto-skip toast 10 секунд** — если все тайлы уже в кэше, показывается ненавязчивый тост «✓ Тайлы в кэше», через 10 с — авто-переход дальше. Раньше таймаут 1.5 с был слишком коротким (экран мелькал). Карта под тостом остаётся интерактивной — можно успеть осмотреться или выбрать область вручную.
- **Прогрев компаса со времени выбора цели** — слушатель курса (`startHeading`) запускается в `App.tsx` как только выбрана цель, а не при монтировании RideScreen. Магнитометр успевает откалиброваться пока идёт кэширование тайлов — к старту PRE_RIDE компас уже тёплый, и course-up карта ориентируется верно с первого кадра. Раньше холодный магнитометр на старте давал кривую ориентацию первые секунды.

### Добавлено

- **ROADMAP.md** — полная дорожная карта развития до v2.0 с монетизацией (Honest Freemium, $2.99 one-time), фичами по версиям (Quick Targets, GPX, 3D Audio, Multi-waypoints, Live Share), шагами публикации в Google Play через TWA, рисками и приоритетами ближайшего спринта.

## [0.5.6] — 2026-05-12

### Производительность

- **Heading throttle** (`src/lib/orientation.ts`): `deviceorientation` событие летело ~60 Hz, на каждом срабатывал `setHeading` → re-render всего RideScreen + пересчёт `courseHeading` useMemo (O(n) walk по треку). Теперь handler вызывается ≤10 Hz и только при дельте ≥2°. Самый большой выигрыш — сотни trig-операций в секунду уходят.
- **`ridden` incremental accumulator**: вместо O(n) прохода по всему треку при каждом re-render (`useMemo([trail])`) — дельта считается в GPS-callback и аккумулируется в ref. O(1) на фикс.
- **`map.easeTo` → `map.jumpTo`** в auto-follow: 800ms анимация прерывалась каждым GPS-тиком (1 Hz), MapLibre дёргал tile/label re-render → GPU гонял зря. Прямой `jumpTo` даёт ту же визуальную плавность (фиксы и так регулярные).
- **Trail coords buffer**: вместо `trail.map(p => [p.lng, p.lat])` (2000 новых nested arrays каждый тик → GC pressure) держим один Array<[lng,lat]> в ref, аппендим только новые точки.
- **Стабильный auto-save interval**: пересоздавался каждый GPS-фикс (`trail`/`time` в deps) → JSON.stringify никогда не достигал интервала. Теперь read-через-ref, interval стабилен.
- **Voice loop стабильный**: `me` в deps пересоздавал `setInterval` каждую секунду → периодическое озвучивание никогда не срабатывало (interval сбрасывался до того как `intervalSec * 1000` истечёт). Заменили на стабильный флаг `hasFix` (false → true один раз).
- **Workbox cache size**: добавлен `maximumFileSizeToCacheInBytes: 5MB`. MapLibre worker chunks (>2MB дефолт) больше не выпадают из precache → офлайн не ломается.

## [0.5.5] — 2026-05-12

### Добавлено

- **4-фазный ride state machine** (`src/lib/rideStateMachine.ts`) — чистый reducer без React. Четыре фазы: `PRE_RIDE` (стоянка до старта), `RIDING` (езда), `SHORT_STOP` (светофор, ≤3 мин), `LONG_STOP` (магазин, обед). Переходы определяются скоростью, расстоянием от якоря и подтверждением подряд N фиксов — фильтрует GPS-джиттер.
- **Smoothed bearing** (`smoothedBearingFromTrail`) — вектор курса берётся не по 2 последним точкам, а от точки ~40 м назад по треку до текущей позиции. Устойчив к шуму GPS.
- **Голосовые сигналы при смене фазы**: «Поехали!» при старте (PRE_RIDE/LONG_STOP → RIDING), отложенная озвучка через 3 с при возобновлении (SHORT_STOP → RIDING).

### Изменено

- **Таймер** паузится в PRE_RIDE и LONG_STOP — время простоя не считается в общее время поездки.
- **Info strip** показывает «⏳ ожидание» вместо скорости/дистанции/времени в PRE_RIDE и LONG_STOP.
- **Голос и обратный таймер** молчат в PRE_RIDE и LONG_STOP.
- **Стрелка «я»** в RIDING — по сглаженному вектору движения, в SHORT_STOP — заморожена, в PRE_RIDE/LONG_STOP — по компасу.
- **Arrival detection** срабатывает только в RIDING — GPS-джиттер на стоянке не вызовет ложное прибытие.
- **Запись трека** только в RIDING и SHORT_STOP (не в PRE_RIDE/LONG_STOP — ходьба по магазину не пишется).
- Плавная анимация стрелки 500 мс при переключении фазы.

## [0.5.1] — 2026-05-12

### Исправлено

- **Голос по-прежнему не работал при выключенном экране** — корень проблемы: `el.play()` вызывался из `useEffect`, который выполняется вне контекста пользовательского жеста, и браузер блокировал воспроизведение. Исправлено: `initWakeAudio()` (создание элемента) теперь вызывается при старте App, а `resumeWakeAudio()` (фактический `play()`) вызывается синхронно внутри обработчика кнопки «Старт →» — это настоящий жест. Дополнительно: слушатель `touchstart/mousedown` на `document` подхватывает первый тап автоматически.
