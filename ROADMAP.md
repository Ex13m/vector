# Vector — Дорожная карта развития

> Голосовой вело-маяк: навигация по часовому циферблату  
> Версия: 0.5.11 | Май 2026

---

## 1. Видение и позиционирование

### Что такое Vector?

Vector — PWA-приложение для велосипедистов, работающее как голосовой маяк. Вместо пошаговых инструкций («через 200 м поверните налево») показывает направление на цель в формате часового циферблата: «цель на 3 часа», «цель на 10 часов». Велосипедист сам выбирает маршрут, исследует город и не зависит от карты на экране.

### Ключевое отличие

Навигация по маяку (beacon), а не по маршруту (route). Велосипедист всегда знает где цель и сколько до неё, но сам решает как ехать.

### Анти-фичи (чего мы НЕ делаем)

- **Нет turn-by-turn:** мы не строим маршрут и не диктуем повороты.
- **Нет социальной ленты:** никаких лайков, комментариев, лидербордов.
- **Нет подписки:** одноразовая оплата или бесплатно.
- **Нет рекламы:** никогда.
- **Нет внешних датчиков:** только телефон.

### Анализ рынка

Ниша голосовой навигации по часовому циферблату полностью пуста. Единственный конкурент в концепции маяка — Beeline:

|                | Beeline                | Vector                          |
|----------------|------------------------|---------------------------------|
| Тип            | Физическое устройство  | PWA-приложение                  |
| Цена           | $99 + $5/мес           | Бесплатно / $2.99 PRO           |
| Индикация      | LED-компас на руле     | Голос + экран + вибрация        |
| Оффлайн        | Только через Bluetooth | Кэшированные тайлы + SW        |
| Голос           | Нет                    | Часовой циферблат + дистанция   |

Конкуренты в маркетах (Komoot, Ride with GPS, OsmAnd, Locus Map) — все работают по модели turn-by-turn и требуют постоянного взгляда на экран. Vector занимает собственную нишу: eyes-free, hands-free вело-навигация.

## 2. Текущее состояние (v0.5.11)

### Работающие функции

- Выбор цели на карте или поиском (Nominatim)
- Кэширование тайлов для оффлайн-работы
- 4-фазный цикл поездки: PRE_RIDE → RIDING → SHORT_STOP → LONG_STOP
- Голосовые объявления: часовое направление, дистанция, повороты, прибытие
- Режим наведения (TargetingHud): компасная стрелка + голос «цель впереди»
- Хим: звуковой сигнал при совпадении курса с вектором на цель (±5°)
- Настройки: язык, голос, единицы измерения, вибрация, интервалы объявлений
- PWA: установка на домашний экран, Service Worker, оффлайн-шелл

### Технологический стек

- React 18 + TypeScript + Vite 5
- MapLibre GL JS (открытый vector tiles)
- Web Speech API (голос)
- DeviceOrientation API (компас)
- Geolocation API (координаты)
- Workbox / vite-plugin-pwa (Service Worker)
- idb-keyval (кэш тайлов, настройки)
- Хостинг: Netlify

## 3. Дорожная карта

### v1.0 — Стабильный релиз + Google Play

**Цель:** довести качество до продакшн-уровня и опубликовать в Google Play через TWA.

- **Полевые испытания.** Тесты на реальных поездках (Android + iOS). Проверка компаса на разных устройствах (deviceorientationabsolute vs webkitCompassHeading). Проверка GPS-точности под деревьями и между зданиями. Проверка чима ±5° и отзывчивости стрелки (15 м сглаживание).
- **Багфиксы по результатам тестов.** Устранение всех найденных проблем: edge-cases GPS, компас, голосовой движок, энергопотребление, screen wake lock.
- **TWA-оболочка.** Trusted Web Activity — способ опубликовать PWA в Google Play без нативного кода. Требуется: Digital Asset Links (файл .well-known/assetlinks.json), подпись APK, аккаунт разработчика Google Play ($25 одноразово).
- **Политика конфиденциальности.** Документ Privacy Policy (требуется для Google Play). Vector не собирает данные — всё хранится локально.
- **Листинг.** Store listing: скриншоты, описание, иконка 512×512, feature graphic 1024×500.

### v1.1 — Quick Targets + PRO

**Цель:** монетизация + удобство повторных поездок.

- **Избранные цели (Quick Targets).** Сохранение часто используемых точек (дом, работа, парк) с быстрым запуском в один тап. Бесплатно: 3 избранных, PRO: без ограничений.
- **GPX-экспорт (PRO).** Запись GPS-трека во время поездки и скачивание в формате .gpx. Полезно для анализа маршрута в Strava, Komoot или других сервисах. Формат: XML-файл с координатами, временем, высотой (если доступна).
- **PRO-разблокировка ($2.99).** Одноразовая покупка. Механизм: покупка через Google Play Billing API (TWA) или Stripe (для PWA-версии). Флаг isPro хранится локально; нет серверной валидации (принцип доверия пользователю).
- **Все слои карт (PRO).** Разблокировка дополнительных слоёв: спутник, рельеф, велодорожки.
- **Выбор голоса (PRO).** Полный список системных TTS-голосов для объявлений.

### v1.2 — 3D Audio (эксперимент)

**Цель:** пространственный звук для интуитивного направления.

- **Web Audio + HRTF.** Использование Web Audio API PannerNode с HRTF-моделью для пространственного позиционирования звука. Цель «звучит» слева/справа/сзади в наушниках.
- **Ограничения.** Требуются наушники (не все велосипедисты ездят в них). Поэтому — экспериментальная PRO-фича, опциональная.
- **Реализация.** AudioContext + PannerNode('HRTF') + AudioListener. Позиция listener = текущая GPS-точка, orientation = courseHeading. Позиция источника = цель.

### v1.3 — Мульти-маршрут (Waypoints)

**Цель:** цепочка точек для сложных маршрутов.

- **Цепочка точек.** Вместо одной цели — массив waypoints. При приближении к точке (<50 м) автоматически переключается на следующую с голосовым объявлением.
- **Редактор точек.** Добавление/удаление/перетаскивание waypoints на карте. Показ номеров и пунктирных линий между точками.
- **Сохранение маршрутов.** Маршруты сохраняются в IndexedDB для повторного использования.

### v1.4 — Статистика поездок

**Цель:** лёгкая аналитика без социальной составляющей.

- **История поездок.** Список завершённых поездок с датой, дистанцией, временем, средней скоростью.
- **Мини-карта трека.** Превью пройденного маршрута на карте (polyline).
- **Недельная/месячная сводка.** Простые графики активности (км/неделю, количество поездок).

### v2.0 — Live Share

**Цель:** отправка своей позиции другу в реальном времени.

- **Ссылка на позицию.** Генерация короткой ссылки, по которой друг видит вашу позицию на карте в реальном времени. Требует минимальный бэкэнд (WebSocket relay).
- **Друг как цель.** Возможность навигировать на движущуюся цель (другого велосипедиста).
- **Приватность.** Ссылка истекает через N часов, нет аккаунтов, нет истории на сервере.

## 4. Монетизация: Honest Freemium

### Принципы

1. Основная навигация бесплатна навсегда.
2. PRO = $2.99 одноразово, не подписка.
3. Никакой рекламы, никогда.
4. Никакого сбора данных, никогда.

### Бесплатно vs PRO

| Функция              | Бесплатно | PRO ($2.99) |
|----------------------|-----------|-------------|
| Навигация по маяку   | ✓         | ✓           |
| Голосовые объявления | ✓         | ✓           |
| Оффлайн-карты       | ✓         | ✓           |
| Избранные цели       | 3 цели    | Без лимита  |
| GPX-экспорт          | —         | ✓           |
| Все слои карт        | —         | ✓           |
| Выбор голоса         | —         | ✓           |
| 3D Audio (HRTF)      | —         | ✓           |
| Multi-waypoints      | —         | ✓           |
| Статистика           | —         | ✓           |

### Механизм оплаты

- **Google Play (TWA):** Google Play Billing API. Покупка «in-app product» типа «one-time». Google забирает 30% (или 15% для малого бизнеса).
- **PWA (web):** Stripe Checkout. Прямая оплата картой, комиссия ~3%. Альтернатива: Gumroad, Paddle.
- **App Store (Capacitor):** StoreKit 2. Apple забирает 30%/15%.

Флаг `isPro` хранится в idb-keyval локально. Принцип доверия: нет серверной валидации, нет DRM. Если кто-то обойдёт — не страшно, фокус на честных пользователях.

## 5. Подготовка к публикации

### Этап 1: PWA (уже работает)

Vector уже доступен как PWA на Netlify. Пользователь открывает URL → «Добавить на домашний экран» → приложение работает fullscreen, оффлайн.

- Стоимость: бесплатно (Netlify free tier)
- Плюсы: мгновенные обновления, нет ревью магазина
- Минусы: нет в маркете, нельзя использовать in-app purchases

### Этап 2: Google Play через TWA

Trusted Web Activity — Android-оболочка, которая открывает PWA в Chrome без адресной строки, как нативное приложение.

**Шаги:**

1. Зарегистрировать Google Play Developer Account (**$25 одноразово**)
2. Создать Digital Asset Links: файл `/.well-known/assetlinks.json` на сайте, связывающий домен с подписью APK
3. Сгенерировать TWA-проект через Bubblewrap CLI или PWABuilder.com
4. Подписать APK/AAB ключом разработчика
5. Подготовить материалы листинга: иконка 512×512, feature graphic 1024×500, минимум 4 скриншота, описание на EN/RU
6. Написать Privacy Policy (обязательно для Google Play)
7. Загрузить AAB в Google Play Console → Internal Testing → Closed Testing → Production
8. Пройти ревью Google (обычно 1–3 дня для новых приложений)

### Этап 3: App Store через Capacitor (позже)

Для iOS нужна нативная оболочка. Capacitor (Ionic) оборачивает веб-приложение в WKWebView с доступом к нативным API.

- Apple Developer Program: $99/год
- Требуется macOS для сборки (Xcode)
- Ревью строже: 3–7 дней
- Можно отложить до v1.4+

## 6. Предложения по улучшению

### Навигация и UX

- **Адаптивное сглаживание стрелки.** Сейчас фиксировано 15 м. Можно привязать к скорости: на 20 км/ч = 15 м, на 40 км/ч = 30 м. Это уберёт дёрганье на скорости без задержки на малой.
- **Умные интервалы объявлений.** При приближении к цели (<500 м) чаще объявлять направление и дистанцию.
- **Wake Lock.** Screen Wake Lock API — не давать экрану гаснуть во время поездки.
- **Авто-остановка.** Автоматическое завершение поездки при прибытии (<30 м от цели) с голосовым объявлением и вибрацией.

### Компас и датчики

- **Калибровка компаса.** Подсказка пользователю при низкой точности магнитометра: «покрутите телефон восьмёркой».
- **GPS-фильтрация.** Отсекать GPS-точки с accuracy > 30 м (в тоннелях, между высотками).
- **Гибридный heading.** На скорости >10 км/ч использовать GPS-bearing (он стабильнее), на месте — компас.

### Энергоэффективность

- **Снижение GPS-поллинга.** В SHORT_STOP/LONG_STOP уменьшать частоту GPS-запросов.
- **Отключение компаса в RIDING.** В движении компас не нужен (курс по GPS). Отключение экономит батарею.

### Доступность (Accessibility)

- **ARIA-метки.** Экранные читалки (TalkBack, VoiceOver) должны мочь озвучить все элементы.
- **Контрастность.** Проверить контрастность HUD-элементов на солнце (WCAG AA минимум).

## 7. Технический долг

- **Тесты.** Нет юнит-тестов. Нужны хотя бы для `geo.ts` (расчёты расстояний/bearing), `orientation.ts` (фильтры), `voice.ts` (форматирование).
- **RideScreen сложность.** Файл растёт. Вынести хуки (`useRidePhase`, `useVoiceAnnounce`, `useCompass`) в отдельные файлы.
- **CI/CD.** Нет конфига CI. Нужно: lint + typecheck + build на каждый push (GitHub Actions).
- **Ошибки.** Нет error boundary и error tracking. Минимум: React ErrorBoundary + Sentry free tier.
